# useState and useEffect Hooks

We have worked with these quite a bit already. So, this is a summary of the key points. When you call the useState method to create the variable and the function to update the variable there are three variations of the method that let us provide different default values.

import { useState } from 'react';

function App() {
  const [data1, setData1] = useState(); //default value will be null
  const [data2, setData2] = useState({ id: 123 }); //default value is the object
  const [data3, setData3] = useState(() => {
    return { now: Date.now() };
  }); //function calculates the default value for the variable
}

The value of a state variable will be maintained across multiple renderings of the component.

The changing of a state variable value will trigger a re-rendering of the component.

Re-rendering a component means calling the component function again.

The useEffect method allows us to access the life cycle of the component. It will accept two parameters, a function and a dependency array. The dependency array can contain reactive values. Props, state, and variables declared inside your component’s body are called reactive values.

import { useEffect } from 'react';

function App({ someProp }) {
  const [data, setData] = useState();

  useEffect(() => {
    //this function runs only when the component is added to the DOM
  }, []);

  useEffect(() => {
    //this function runs when the component is added to the DOM
    //AND
    //when either data or someProp get new values.
  }, [data, someProp]);

  useEffect(() => {
    //runs when first rendered
    let interval = setInterval(() => {
      //this function runs every two seconds roughly
      console.log('interval tick');
    }, 2000);
    return () => {
      //this is the clean up function
      //it runs when the component is removed from the DOM
      clearInterval(interval);
    };
  }, []);
}

We can use as many useEffect functions as we want in our component.

A lesser known feature of the useEffect function is a clean up function. Any useEffect function can return a function, which will be the clean up function. When the component is removed from the DOM, this clean up runs. Its purpose is to stop timers, clear promises, stop fetches, etc.

# useRef Hook

React has had the concept of references, or refs, for a while. Originally they were specifically designed to hold references to DOM elements so they can be used inside functions. In version 16 of React, when hooks were added, the useRef hook was added as the recommended way to create references. In version 18 of React we gained the ability for refs to hold any value, not just DOM elements.

The purpose of a ref now is to hold ANY value that we want to survive across multiple renders of a component.

The difference between a ref and a state variable is that changing a ref will NOT trigger a new render of the component.

import { useRef } from 'react';

function App() {
  const myRef = useRef(0); //zero is the default starting value

  function handleClick() {
    //called when clicking on the button
    let num = Math.floor(Math.random() * 100) + 1; // 1 - 100
    myRef.current = num;
    console.log('New value of ref', myRef.current);
  }

  return (
    <p>
      <button onClick={handleClick}>Click to update Ref</button>
    </p>
  );
}

# useMemo Hook

The useMemo hook lets React cache the result of a function call. The idea is that running a function takes time and processing power. So, if you are going to be calling a function a whole bunch of times, and the function returns the same result when given the same input parameters, then we can save the result of the function. The useMemo hook will memoize the function. This means that you run the function with a set of parameters and save the results to skip running it again in the future.

import { useMemo } from 'react';

function List({ todos, id }) {
  //todos - the list of objects
  //id - the id for an item to remove

  const visibleTodos = useMemo(() => filterTodos(todos, id), [todos, id]);

  function filterTodos(_todos, _id) {
    return _todos.filter((t) => t.id != _id);
  }

  return (
    <ul>
      {visibleTodos.map((item) => {
        return <li key={item.id}>{item.name}</li>;
      })}
    </ul>
  );
}

# useCallback Hook

The useCallback hook is very similar to the useMemo hook. Both will try to optimize performance by caching things.

The primary difference is the useMemo caches the Result of a function based on the parameter values being passed to it. The useCallback hook caches the actual function itself based on the parameter values.

function App() {
  //instead of this
  const handleClick = () => {
    console.log('Button clicked!');
  };
}

//do this
import { useCallback } from 'react';

function App() {
  const handleClick = useCallback(() => {
    console.log('Button clicked!');
  }, []);
  //handleClick is now the cached version of the function
}

# useId Hook

useId is a React Hook for generating unique IDs that can be passed to accessibility attributes. Like ALL hooks, you can’t call it inside loops or conditions. This means that you can't call it inside a loop to generate a series of ids.

import {useId} from 'react';

function App(){
  const uuid = useId();

  function processForm(formData){
    //handling a submitted form to create a new user
    formData.append('user_id', uuid);
    //send our user data to an API endpoint
    fetch(url, {
      method: 'POST',
      headers: {
        'content-type': 'multipart/form-data'
      }
      body: formData
    })
  }

  return (
    <form action={processForm}>
      ...
    </form>
  )
}

# useReducer Hook

useReducer is a React Hook that lets you add a reducer to your component. It has a signature of useReducer(reducer, initialArg).

import { useReducer } from 'react';

function reducer(state, action) {
  //the function that takes a new state value
  //and an action request
}

function MyComponent() {
  const [state, dispatch] = useReducer(reducer, { id: 42 });
  //state - the current state object
  //dispatch - the function to call and pass the new state and action values
}

Back in week 9 there was discussion about the idea of finite state machines, the pub sub design pattern, and the idea of a reducer. This useReducer hook creates a reducer connected to our component, in the same way that a state variable is connected to our component.

Here is an example for a reducer function to be used with the hook. We will use the example of a movie object which has three properties - id, title, and status. The status can be one of the values: IN_CART, RENT_MOVIE, WATCH_MOVIE or RESET. The different actions that can be passed to the reducer function are ADD_TO_CART, RENT_MOVIE, WATCH_MOVIE, or RESET.

export const movieReducer = (state, action) => {
  switch (action.type) {
    case 'ADD_TO_CART':
      return { ...state, status: 'IN_CART', id: action.id, title: action.title };

    case 'RENT_MOVIE':
      if (state.status === 'IN_CART') {
        return { ...state, status: 'RENTED' };
      }
      return state;

    case 'WATCH_MOVIE':
      if (state.status === 'RENTED') {
        return { ...state, status: 'WATCHED' };
      }
      return state;

    case 'RESET':
      return { id: action.id, title: state.title, status: 'SEARCH_RESULTS' };

    default:
      return state;
  }
};

const movieReducer = (state, action) => {
  // ...

export default function Component() {
  const [movie, dispatch] = useReducer(movieReducer, initialState);

  return (
    <div>

# useFormState Hook

The useFormStatus hook will return several properties about the submission of a form. The pending property is true if the form is still being processed. Good for disabling buttons.

The data property is the formData object from the form.

The method property is the method for the form. Eg: POST or GET

The action property is a reference to the function.

import { useFormStatus } from 'react-dom';
import action from './actions';
//action is the async function for processing the form... promise-based

function Submit() {
  const { pending, data, method, action } = useFormStatus();
  return <button disabled={status.pending}>Submit</button>;
}

export default function App() {
  return (
    <form action={action}>
      <Submit />
    </form>
  );
}

# Context API

The Context API can be used for global state management in a React app.

Let's say that we have some data which we need to access from numerous components inside our React app. For example, a selected user preference for a theme plus the user's name and email. We could create a state variable in the <App> component and then pass the state variable as a prop down through all of the other components.

Unfortunately, this approach requires a lot of editing of different components because we have to add multiple props in every component file. It is way to easy to make mistakes or miss one of these.

So, the better approach is to use a Context object at the <App> level as the container for this global state data. This global state Context object has two main parts - a Provider which holds the value of the data, and a hook function to access the value of the data.

First step is to create a file for our context object in a context folder.

Let's create a context object for a theme in this file - /src/context/theme-context.js.

Our file needs to create a context object using the createContext method. The context object will contain a Provider object. So, we need to have a function that will return the Provider.

import { createContext, useState, useContext } from 'react';

const ThemeContext = createContext();
//create the context object

function ThemeProvider(props) {
  //the function to create the provider

  return <ThemeContext.Provider {...props} value={{ color: rebeccapurple }} />;
}

export { ThemeProvider };

Now, to be able to access the value from the Provider, we need to create a hook inside of the /src/context/theme-context.js file.

import { createContext, useState, useContext } from 'react';
const ThemeContext = createContext();

function ThemeProvider(props) {
  //the function to create the provider
  //add any functions and properties we want here
  return <ThemeContext.Provider {...props} value={{ color: rebeccapurple }} />;
}

function useTheme() {
  const context = useContext(ThemeContext);
  //get a reference to the ThemeContext.Provider.value
  //first make sure that the hook is calling from the right place
  if (!context) throw new Error('Hook is not inside the provider');
  return context;
}
export { ThemeProvider, useTheme };

// /src/components/header.jsx
import { useTheme } from '../context/theme-context.jsx';

  const theme = useTheme();
