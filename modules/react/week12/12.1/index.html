# React Router and Remix

React Router is the most common routing and navigation library being used with React. It is closely connected with Remix. React Router was the original library for routing and it was built-in to early versions of React. Then the routing was split off into its own project called React Router. After a few years, the project Remix was created as a full framework on top of React and React Router. Lots of updates were made to the routing through the Remix project.

Version 7 of React Router now includes updates made to the routing process through the Remix project and the latest version 19 of React. In this course, we will be using React Router v7 as the navigation solution for our React projects.

# Building A Router

What is a Router? A Router is a component that represents a specific URL, and when that URL appears in the location bar, it will load a specific UI component. We add a <Routes> component as a wrapper around all the individual <Route> components. The first step is a Router object that wraps around the <Routes> component. The Router object, like <BrowserRouter> controls how the URL is read and interpreted.

The full API reference for React Router (opens new window).

Open the main.jsx file and import Routes, Route, and then BrowserRouter. Later we will talk about HashRouter as an alternative to BrowserRouter. With a BrowserRouter we are looking at a full pathname from the URL. The HashRouter uses the hash part of the path. There are also StaticRouter and MemoryRouter.

import { BrowserRouter, Routes, Route } from 'react-router';

import { createRoot } from 'react-dom/client';
import './index.css';
import App from './App.jsx';

createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <Routes>
      <Route path="/" element={<App />} />
    </Routes>
  </BrowserRouter>
);

# Building Routes

When we create a list of <Route> components, each one needs a path property, which represents part of the path to display in the URL, plus an element property. The element property is the component to load.

In the following example, the following paths will match these components:

  • /: <Home/>
  • /products: <ProductList />
  • /contact: <Contact />
  • /products/234: <Product />
  • /products/678: <Product />
  • /products/456/edit : <EditProduct />
<Routes>
  <Route index element={<Home />} />
  <Route path="products" element={<ProductList />} />
  <Route path="products/:pid" element={<Product />} />
  <Route path="products/:pid/edit" element={<EditProduct />} />
  <Route path="contact" element={<Contact />} />
</Routes>

Two of these routes have a segment of their path, which starts with :. This segment is known as a route param. The useParams hook is how we can extract the actual value of this part of the URL to use in our component. The useSearchParams hook will read the values passed through the querystring.

import { useParams, useSearchParams } from 'react-router';

export default function Product() {
  // let params = useParams();
  const { pid } = useParams();
  // or without destructure params.pid

  let [searchParams, setSearchParams] = useSearchParams();
  //If you want to get the values from the querystring, use the useSearchParams hook
  // the searchParams object is a URLSearchParams object

  return (
    <>
      <h3>Product Details</h3>
      <p>Component for product {pid}.</p>;
    </>
  );
}

It is possible to use multiple dynamic segments within a path.

<Route path="/c/:categoryId/p/:productId" element={<Product />} />

<Route path="users/:userId/edit?" element={<User />} />
1

If we want a path to cover any version of a URL, of any number of segments, that starts with a specific value, then we can use a splat. In the following example, the <File /> component will handle any of these example paths:

  • /files
  • /files/images
  • /files/images/png
  • /files/audio/mp3/green-day
<Route path="files/*" element={<File />} />
1

Inside the <File /> component we can access the value of the URL that comes after /files/ with this:

let { '*': splat } = useParams();
//extract the "*" value from the URL and rename the variable as "splat"
console.log(splat);
//"" or "images" or "images/png" or "audio/mp3/green-day"

# Nested Routes

When we nest one <Route> inside another, it means that the outer route will act like a layout container for the inner one. The outer route needs to have an <Outlet/> component. This outlet gets used as the target location to load the inner component. The inner component is selected based on the path.

<Routes>
  <Route path="dashboard" element={<Dashboard />}>
    <Route index element={<Home />} />
    <Route path="settings" element={<Settings />} />
    <Route path="*" element={<NotFound />} />
  </Route>
</Routes>

The path of the parent is automatically included in the child, so this config creates both "/dashboard" and "/dashboard/settings" URLs. The index property represents the default route based on the outer route's path.

Child routes are rendered through the <Outlet/> in the parent route. With the above Routes, the Dashboard component will look like this:

import { Outlet } from 'react-router';

export default function Dashboard() {
  return (
    <div>
      <h1>Dashboard</h1>
      {/* Outlet will load either <Home/> or <Settings/> */}
      <Outlet />
    </div>
  );
}

# Navigating

With React Router handling the routes, we can use the <Link> or <NavLink> components to create our anchor tags in our pages. The difference between these two components is that the NavLink has the ability to indicate a match with the current path.

import { Link, NavLink } from 'react-router';

function Header() {
  return (
    <nav>
      {/* NavLink makes it easy to show active states */}
      <NavLink to="/" className={({ isActive }) => (isActive ? 'active' : '')}>
        Home
      </NavLink>
    </nav>
  );
}

# useNavigate Hook

There WILL be situations when a <Link> or a <NavLink> does not meet our needs. We need to be able to programmatically navigate from one Route to another. Maybe the user is going to click on a button that carries out some async task, and after the task is complete, we want to start the navigation.

To achieve this, we will use the useNavigate hook, which will give us a navigate method that we can use to navigate between routes whenever we want.

import { useNavigate } from 'react-router';
import MyLoginForm from './components/myloginform';

export function LoginPage() {
  let navigate = useNavigate();

  return (
    <main>
      <MyLoginForm
        onSuccess={async () => {
          await processForm();
          navigate('/dashboard');
        }}
      />
    </main>
  );
}

# Hook Version of Routes

We can also use the useRoutes hook inside of <App> to define the routes for our application, instead of using the <Routes> and <Route> components in the JSX.

The following example will create the routes:

  • / will load <Home>
  • /about will load <About>
  • /products will load <ProductList> inside an <Outlet> in <Home>
  • /tasks will load <TaskList> inside an <Outlet> in <Home>
import * as React from 'react';
import { useRoutes } from 'react-router';

function App() {
  let element = useRoutes([
    {
      path: '/',
      element: <Home />,
      children: [
        {
          path: 'products',
          element: <ProductList />,
        },
        { path: 'tasks', element: <TaskList /> },
      ],
    },
    { path: 'about', element: <About /> },
  ]);

  return element;
}

# Why BrowserRouter or HashRouter

The different types of Routers that are part of React Router let us manage the navigation in different ways. It can use the pathname part of the URL or the hash part of the URL.

If you use the <BrowserRouter> as the routing provider at the top of the App, then it means that we will be using the pathname part.

If you use the <HashRouter> as the routing provider at the top of the App, then it means that we will be using the hash part.

# TanStack

TanStack Router is an alternative routing and navigation solution for React. TanStack Router website (opens new window).

TanStack also has another library called TanStack Query which can be used for caching server api results to help with app performance. TanStack Query website (opens new window)

For TanStack routing we will start by using the tanstack create command to create the React project with the TanStack navigation capabilities. It will use TypeScript over JavaScript. Start with:

npm create @tanstack/router@latest
1

It will prompt your for a project name, the bundler (which should be vite), the IDE (which should be VSCode), and if you would like it to open the project with the IDE.

Once the project is created and opened in the IDE, there are a few important files and folders for us to use.

The initial file for the React app will be src/main.tsx. This is what it will be.

import React from 'react';
import ReactDOM from 'react-dom/client';
import { RouterProvider, createRouter } from '@tanstack/react-router';
import { routeTree } from './routeTree.gen';

// Set up a Router instance
const router = createRouter({
  routeTree,
  defaultPreload: 'intent',
});

// Register things for typesafety
declare module '@tanstack/react-router' {
  interface Register {
    router: typeof router;
  }
}

const rootElement = document.getElementById('app')!;

if (!rootElement.innerHTML) {
  const root = ReactDOM.createRoot(rootElement);
  root.render(<RouterProvider router={router} />);
}

The router object being passed to the router property inside the RouterProvider is being created by the createRouter method. To create the actual routes, we simply have to create files inside the /src/routes/ folder.

The __root.tsx file will define the layout that will wrap around every page. The following code snippet shows the contents for the __root.tsx file. The <Outlet/> element is where each of the route files will be loaded.

import * as React from 'react';

export const Route = createRootRoute({

The NEW part is the call to the createFileRoute method. This method takes the route that the file represents plus an object with a component property that points to the React component for the page.

To learn more about how TanStack Router does file matching read this reference (opens new window)

The file src/routes/posts.tsx will match the route /posts. The file src/routes/posts/index.tsx will work as an exact only match for the route /posts. If we want to pass dynamic values through the URL and match a specific file, then we build a file whose name starts with $. The file src/routes/posts/$postId.tsx will match against /posts/$postId, where $postId stands in as a variable representing any value in the URL.
