# Setting Up for Tailwind in React

The following installation and configuration notes are for Version 4 of Tailwind, in a Vite controlled, plain React project.

  1. First we need to create our React project with vite, as per normal.
npm create vite@latest project-name
npm install
1
2
  1. Install Tailwind CSS and its Peer Dependencies:

Second, we'll need to install Tailwind CSS, and the Vite package for tailwindcss as development dependencies. Open your terminal in your project directory and run:

npm install tailwindcss @tailwindcss/vite
1
  1. Configure Plugin Paths:

The vite.config.js file should have the following contents. It needs to call both the react and tailwind plugins.

import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react-swc';
import tailwindcss from '@tailwindcss/vite';

// https://vite.dev/config/
export default defineConfig({
  plugins: [react(), tailwindcss()],
});

  1. Add Tailwind Directives to Our CSS:

Create a global CSS file (or use an existing one) and add the Tailwind CSS directives. Typically this is done in the index.css file. If our project does not have one create it in the src folder. This is the CSS file imported by main.js. We can remove much of the default styling so we are just left with:

/* ./src/index.css */
@import 'tailwindcss';
/* formerly this was a series of @tailwind commands */

:root {
  color-scheme: light dark;
  font-synthesis: none;
  text-rendering: optimizeLegibility;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

body {
  margin: 0;
  min-width: 320px;
  min-height: 100vh;

# Functions and Directives

There are a series of commands that we can add to our .css files when working with tailwind.

@import tailwind will include the TailwindCSS library in your project.

@theme { } will let us create a series or variables that we can use through Tailwind classes. In this directive block, we can define design tokens, fonts, colors, and break points.

@theme {
  --font-display: 'Merriweather', 'sans-serif';
  /* custom font stack to be used like className="font-display" */

  --color-almost-black: #020202;
  --color-ultra-black: #010101;
  /* custom colors to be used with text- and bg-. eg: className="bg-ultra-black" */

  --ease-snappy: cubic-bezier(0.2, 0, 0, 1);
  /* custom ease to be used with transitions */

  --breakpoint-3xl: 120rem;
  /* custom breakpoint to be used like: className="3xl:grid-cols-6 grid grid-cols-2" */

  --text-xxxxl: 40rem;
  /* for creating or setting a font size to use like: className="text-xxxxl" */

  --spacing-5: 8rem;
  /* for creating a numeric value to use for padding, margins, heights, widths: className="px-5 my-5 max-h-5 w-5" */
}

# Material UI

TailwindCSS is a great product which lets you define your own styles through variables and css class names. To use this effectively you need to have a design system planned.

There are also, however, UI Kits which create responsive components that all use the same design system which is already defined. Material UI is one of these UI Kits. It is a set of values that apply the Material Design design system to the components. Material UI website (opens new window). (There are other UI Kits too).

You would not use TailwindCSS AND Material UI on the same project. You would use Material UI instead.

# Installation

Once you have created your Vite-React project and installed the dependencies,

npm install @mui/material @emotion/react @emotion/styled
1

Material UI uses the Roboto font by default. We can add that to our project by running this install command.

npm install @fontsource/roboto

# Usage

Once you have added Material UI to your React App, you can start building your components with it. Let's say that we want to have a custom button, whose label will get passed through props to the component.

//
import Button from '@mui/material/Button';

export default function ButtonUsage({ label, size = 'Medium', color = 'Primary', action }) {
  //size values are Small, Medium, or Large
  //color values are Success, Primary, Secondary, Error, Warning, Info
  return (
    <Button variant="contained" size={size} color={color} onClick={action}>
      {label}
    </Button>
  );
}

# Theme Providers

Remember how we build a Context object and then wrapped its Provider around the <App/> earlier?

Well we can do the same thing and create a ThemeProvider with a custom color scheme for Material UI.

Start by creating a Theme object that can be passed to the Provider. This can be in its own theme.js file or inside the main.jsx file. Then we pass the theme object that we create to the Provider that wraps around <App/>.

import { createTheme, ThemeProvider } from '@mui/material/styles';
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

const theme = createTheme({
  palette: {
    primary: {
      main: '#FF5733',
      // light: will be calculated from palette.primary.main,
      // dark: will be calculated from palette.primary.main,
      // contrastText: will be calculated to contrast with palette.primary.main
    },
    secondary: {
      main: '#E0C2FF',
      light: '#F5EBFF',
      // dark: will be calculated from palette.secondary.main,
      contrastText: '#47008F',
    },
    spacing: (factor) => `${0.25 * factor}rem`,
    typography: {
    fontFamily: [ /** custom font stack **/ ],
  },
});

ReactDOM.createRoot(document.getElementById('root')).render(
  <ThemeProvider theme={theme}>
    <App />
  </ThemeProvider>
);

# Responsive Layout

Here is the Guide to building Responsive Layouts with Material UI (opens new window)

With Material UI, in addition to the components you can also create layouts. There are a number of components specifically for creating your layout. These include <Grid>, <Box>, <Stack>, and <Container>.

The Container centers your content horizontally. It's the most basic layout element.

The Box component is a generic, theme-aware container with access to CSS utilities from MUI System. It is a generic container for grouping other components.

The Stack is a container component for arranging elements vertically or horizontally. Think column or row.

The Grid component works well for a layout with a known number of columns. The columns can be configured with multiple breakpoints to specify the column span of each child.
